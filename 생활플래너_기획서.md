# 생활 플래너 앱 기획서

> 프로젝트명: 미정 | 작성일: 2026년 6월
> 표기: ✅ 확정 / ⏸ 보류 / ❌ 제외

---

## 1. 프로젝트 개요

계획(Plan) – 실행(Do) – 측정(Measure) – 기록(Log)을 하나의 흐름으로 통합한 생활 플래너 앱이다. 사용자는 일/주/달/년 등 원하는 기간 단위로 일정을 자유롭게 설계하고, 실제로 그 활동에 얼마나 집중했는지를 자동으로 측정하며, 그 결과를 시각적으로 기록하고 돌아볼 수 있다.

'잠든 사이 숲' 프로젝트와는 무관한 별도의 독립 프로젝트이다.

---

## 2. 배경 및 문제 정의

기존의 일정 관리·투두 앱은 "할 일을 등록하고 체크하는" 수준에 머무르는 경우가 많다. 일정을 세웠다는 것과 실제로 그 시간만큼 집중했다는 것은 별개의 문제인데, 이 둘 사이의 간극을 추적하고 보여주는 기능은 부족한 편이다.

본 프로젝트는 다음 두 가지 문제의식에서 출발한다.

- 계획은 세우지만 실제로 얼마나 지켰는지 알기 어렵다 — 계획 대비 실제 집중 시간을 비교할 방법이 마땅치 않음
- 기록이 번거롭다 — 활동을 시작/종료할 때마다 폰을 직접 조작해야 하는 번거로움 때문에 측정을 포기하게 됨

이를 해결하기 위해, 화면 on/off 같은 단순한 신호를 활용해 사용자의 수동 조작을 최소화하면서도 정확하게 집중 시간을 측정하고, 계획 대비 달성률을 시각적으로 보여주는 것을 핵심 목표로 삼는다.

---

## 3. 타겟 사용자

- **시간관리/자기관리에 관심이 많은 사람** — 학생, 직장인 등 루틴을 시각적으로 관리하고 싶은 사용자
- **실제 집중 시간이 궁금한 사람** — 공부나 업무에 들인 시간을 "계획"이 아닌 "실제 측정값"으로 확인하고 싶은 사용자
- **꾸준함에 동기부여가 필요한 사람** — 잔디(출석) 형태의 시각적 기록으로 동기를 얻는 "갓생/스터디로그" 트렌드에 관심 있는 사용자

---

## 4. 핵심 차별점

- **블록 기반의 자유로운 커스터마이징** — 색상, 태그, 중첩 구조로 사용자가 직접 설계하는 일정 (정해진 템플릿이 아님)
- **화면 on/off 기반 자동 타이머 제어** — 폰을 보지 않고도 자연스럽게 집중 시간이 측정됨
- **계획 대비 실제 달성률 시각화** — 단순 기록을 넘어 "얼마나 지켰는지"를 퍼센트로 보여줌
- **깃허브 잔디 스타일 출석 기록** — 누적된 노력을 한눈에 보여주는 시각적 동기부여 장치
- **활동 단위의 세부 회고 기록** — 단순 시간 기록이 아니라 "무엇을 했는지"까지 남길 수 있음

---

## 5. 핵심 기능 명세

### [확정] 5.1 계획표 기본 구조

- 일 / 주 / 달 / 년 등 원하는 기간 단위로 설정 가능
- 활동을 블록 형태로 만들어 시간대에 끌어다 배치 (드래그 앤 드롭)
- 블록마다 색상 커스터마이징 가능
- 태그를 통해 관련 있는 활동끼리 묶기 가능

### [확정] 5.2 블록 중첩 구조

- 블록 안에 블록(자식 블록)을 넣을 수 있음
- 자식 블록 형태: 체크리스트형(시간 추적 없음) + 독립 타임블록형(자체 시간·타이머 보유) 둘 다 지원
  - 타임블록끼리의 중첩은 1단계만 허용 (부모→자식까지만) — 캘린더 표시·타이머 합산·시간 겹침 체크의 복잡도를 제한하기 위함
  - 체크리스트는 무제한 중첩 허용 — 시간 추적이 없는 단순 항목이라 구현 부담이 적고, 세분화 요구를 이쪽으로 흡수

### [확정] 5.3 타이머 시스템

- 사용자가 직접 시작 / 정지하며, 실제 집중 시간을 측정
- 상태는 실행중 / 자동 일시정지 / 수동 정지의 3가지로 구분 (화면 on/off와 수동 조작이 충돌하지 않도록)

| 상태 | 진입 조건 | 화면 꺼짐 시 동작 |
|---|---|---|
| 실행중 | 사용자가 시작 버튼을 누름 | 화면이 켜지면 → 자동 일시정지로 전환 |
| 자동 일시정지 | 화면이 켜져서 시스템이 자동으로 멈춤 | 화면이 꺼지면 → 자동으로 실행중 복귀 |
| 수동 정지 | 사용자가 직접 정지 버튼을 누름 | 화면 on/off와 무관하게 반응 없음 — 재시작은 사용자가 직접 |

핵심: "화면 꺼짐 → 재시작" 규칙은 자동 일시정지 상태에서만 발동하며, 수동 정지 상태에서는 절대 발동하지 않는다. (예: 화장실에 가며 수동으로 끄고 간 경우, 화면이 잠겨도 자동 재시작되지 않음)

- 방치 알림: 수동 정지 상태가 일정 시간(사용자 설정 가능) 이상 지속되면 "타이머가 N분째 멈춰있어요" 식으로 알림 발송 — 재시작을 깜빡하는 것을 방지
- 자동재시작 시 확인 알림(컨펌 팝업)은 두지 않음

#### [보류] 5.3.1 가로모드 상시 디스플레이

- 화면을 계속 켜두고 타이머를 표시하는 기능 — 배터리 소모, "폰 안 보고도 측정"이라는 핵심 가치와의 상충, 산만함 유발 가능성 때문에 보류
- 도입 시에는 "화면 on/off"가 아닌 "타이머 디스플레이 화면에 머무르는지"를 기준으로 자동정지·재시작 로직을 재정의해야 함

### [확정] 5.4 뽀모도로 모드

- 온/오프 토글 가능
- 공부시간 / 쉬는시간을 사용자가 직접 설정 (고정 25/5분이 아님)
- 설정한 시간 도달 시 알림

### [확정] 5.5 위젯 & 잠금화면

- 위젯: 현재 시간대에 설정된 활동 표시
- 잠금화면: 타이머 노출, 정지/완료 버튼으로 직접 조작 가능
- 다음 활동까지 남은 시간 표시는 제외

### [확정] 5.6 출석 체크 (잔디)

- 활동 완료 체크 또는 설정 시간 이상 집중 시 깃허브 잔디 스타일로 기록
- 잔디 클릭 시 해당 날짜의 활동 상세 보기 가능
- 스트릭 프리즈, 태그별 레벨 시스템은 도입하지 않음

### [확정] 5.7 목표 & 통계

- 오늘 기준 체크리스트 달성률(%) 시각화
- 시간 목표 대비 실제 기록 시간 비율(%) 시각화
- "아직 끝내지 못한 일" 섹션을 별도로 운영 — 과거에 시간대를 정해 계획했지만 완료하지 못한 블록/활동을 모아두는 곳 ("계획 vs 실제"의 실패 기록)
  - 사용자가 완료 체크하거나 삭제하면 목록에서 사라짐

### [확정] 5.8 반복 & 템플릿

- 반복 설정 기능 (매일/매주 반복되는 블록)
- 하루/주 전체를 템플릿으로 복제하는 기능
- 편의성을 위해 둘 다 도입

### [확정] 5.9 일정 관리

- 블록끼리 시간이 겹치면 경고 표시
- 블록의 시각적 크기는 설정된 시간 길이에 비례하여 캘린더에 표시
- 습관 스태킹: 블록끼리 "이거 끝나면 이거" 식으로 연결해 루틴 체인 구성
- 구글 캘린더 연동은 제외, 아이젠하워 매트릭스 뷰는 보류

### [확정] 5.10 회고 & 기록

- 활동(블록) 단위로 구체적인 메모 작성 가능 (예: "운영체제 공부" 블록에 "7장 페이징, 8장 가상메모리 정리함" 기록)
- 오늘의 기분을 하루 단위로 이모지로 간단히 표현

### [확정] 5.11 마감 작업 (데드라인 태스크)

- 특정 시간대 없이 날짜(마감일)만 지정하는 작업 타입 (예: "기말 레포트 — 6월 30일까지")
- 캘린더 뷰에 마감일에 마커로 표시
- "아직 끝내지 못한 일"(5.7)과는 별개의 독립적인 개념
  - 아직 끝내지 못한 일 = 과거 계획의 실패 기록
  - 마감 작업 = 애초에 시간대를 정한 적 없는 작업
- 마감 작업은 자체적으로 상태를 가짐: 진행중 / 마감 지남(미완료) — 지나도 5.7 섹션으로 합쳐지지 않고 마감 작업 목록 안에서 "지난 마감"으로 별도 표시
- 사용자가 완료 체크하거나 삭제하면 목록에서 사라짐
- 필요하면 사용자가 직접 시간대를 지정해 일반 타임블록으로 전환 가능 (마감 작업 → 타임블록)

## 6. 보류 및 제외 기능

### 보류 (나중에 재검토)

- 컨텍스트 인식 (위치 기반 활동 추천, 백색소음 자동재생)
- 공유 (일정 이미지 SNS 공유, 블록 템플릿 마켓플레이스)
- 기타 (PC 브라우저 확장 연동, 음성 단축어)
- 아이젠하워 매트릭스 뷰
- 가로모드 상시 디스플레이

### 제외 (도입하지 않음)

- 구글 캘린더 연동
- 스트릭 프리즈
- 태그별 레벨 시스템
- 위젯에 다음 활동 남은 시간 표시
- 타이머 자동재시작 시 확인 알림
