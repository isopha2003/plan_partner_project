# 생활 플래너 앱 기획 정리

> 추후 Claude Code로 구현할 때 참조용 문서.
> '잠든 사이 숲'과는 별개의 독립 프로젝트.
> 표기: ✅ 확정 / ⏸ 보류(나중에 재검토) / ❌ 제외(도입 안 함)

---

## 1. 계획표 기본 구조 ✅

- 일 / 주 / 달 / 년 등 사용자가 원하는 기간 단위로 설정 가능
- 블록은 **템플릿(재사용 가능한 활동 정의)**과 **인스턴스(캘린더에 배치된 것)**로 분리됨
  - **블록 템플릿**: 제목 + 색상만 가진 재사용 가능한 활동 정의 (예: "공부", "운동"). 한 번 만들어두고 여러 날 여러 번 끌어다 쓸 수 있음 — 매번 새로 만들 필요 없음
  - **블록 인스턴스**: 블록 템플릿을 캘린더의 특정 시간대에 끌어다 놓아서(드래그 앤 드롭) 생성되는 실제 배치. 날짜·시작/종료 시간·완료 여부를 가짐
  - 템플릿 색상을 바꾸면 그 템플릿을 쓰는 모든 인스턴스에 반영됨
  - **템플릿을 캘린더에 드래그해서 놓으면 시간 입력 화면 없이 즉시 인스턴스 생성** (기본 길이로 배치, 15분 스냅). 이후 시간 조정은 가장자리 드래그(리사이즈)로만 — 별도 시간 피커 입력 단계 없음
- 태그를 통해 관련 있는 활동끼리 묶기 가능 — **템플릿 단위로 부여** (같은 활동 종류는 항상 같은 태그를 가짐)
- 템플릿 팔레트(드래그 시작점)는 평소 가로 스크롤 한 줄로 보이다가, 위로 드래그하면 펼쳐져서 그리드로 더 많은 템플릿을 한 번에 보여주는 확장형 바텀시트 형태

## 2. 블록 중첩 구조 ✅

- 블록 안에 블록(자식 블록)을 넣을 수 있음
- 자식 블록 형태는 상황에 따라 **둘 다 지원**:
  - 체크리스트형: 단순 체크만, 자체 시간/타이머 없음
  - 독립 타임블록형: 자식도 각자 시작/종료 시간 + 타이머 보유, 부모와 마찬가지로 블록 템플릿을 참조
- 중첩 깊이 규칙:
  - **타임블록끼리의 중첩은 1단계만** (부모 타임블록 → 자식 타임블록까지만, 그 이상 불가)
    - 이유: 캘린더 표시, 타이머 합산, 시간 겹침 체크가 단계가 늘수록 기하급수적으로 복잡해짐
  - **체크리스트는 무제한 중첩 허용**
    - 이유: 시간 추적이 없는 단순 텍스트+체크박스라 구현 부담이 적음. "더 세분화해야 하는 상황"을 이쪽으로 흡수

## 3. 타이머 시스템 ✅

### 3.1 기본 동작
- 사용자가 직접 시작 / 정지
- 실제 집중 시간을 측정하는 용도

### 3.2 상태 3가지 (중요 — 자동/수동 정지 충돌 방지)

| 상태 | 진입 조건 | 화면 꺼짐 시 동작 |
|---|---|---|
| 실행중 | 사용자가 시작 버튼 누름 | 화면 켜지면 → 자동 일시정지로 전환 |
| 자동 일시정지 | 화면이 켜져서 시스템이 자동으로 멈춤 | 화면 꺼지면 → 자동으로 실행중 복귀 |
| 수동 정지 | 사용자가 직접 정지 버튼 누름 | 화면 꺼짐/켜짐과 **무관하게 아무 반응 없음**, 재시작은 사용자가 직접 |

- 핵심: "화면 꺼짐 → 재시작" 규칙은 **자동 일시정지 상태에서만** 발동. 수동 정지 상태에서는 절대 발동 안 함
  - 예시 시나리오: 화장실 가면서 수동으로 타이머를 끄고 감 → 화면이 잠겨도 자동 재시작 안 됨 (의도적 정지이므로)
- 자동재시작 시 확인 알림(컨펌 팝업): ❌ 제외

### 3.3 방치 알림 ✅
- 수동 정지 상태가 일정 시간(사용자 설정 가능) 이상 지속되면 알림 발송
- 예: "타이머가 N분째 멈춰있어요" 식으로 리마인드
- 재시작을 깜빡하는 것을 방지하기 위한 용도

### 3.4 가로모드 상시 디스플레이 ⏸
- 화면을 계속 켜두고 타이머를 보이게 하는 기능 → 보류
- 검토 이유: 배터리 소모, 앱의 핵심 가치("폰 안 보고도 자동 측정")와 상충, 화면 켜둠으로 인한 산만함 유발 가능성
- 나중에 도입 시: "화면 켜짐/꺼짐" 대신 "타이머 디스플레이 화면에 머물러 있는지"를 기준으로 자동정지/재시작 트리거를 재정의해야 함 (그래야 가로모드와 기존 자동정지 로직이 충돌 안 함)

## 4. 뽀모도로 모드 ✅

- 온/오프 토글 가능
- 공부시간 / 쉬는시간을 사용자가 직접 설정 (고정 25/5분이 아님)
- 설정한 시간 도달 시 알림

---

## 5. 위젯 & 잠금화면 ✅

- 위젯: 현재 시간대에 설정된 활동을 표시
- 잠금화면: 타이머 노출, 정지/완료 버튼으로 잠금화면에서 직접 조작 가능
- 다음 활동까지 남은 시간 표시: ❌ 제외

---

## 6. 출석 체크 (잔디) ✅

- 활동을 완료 체크하거나, 설정한 시간 이상 집중했을 때 깃허브 잔디 스타일로 기록
- **기록 기준은 활동이 계획됐던 날짜가 아니라 실제로 완료 처리한 날짜** — 예: 1월 1일 활동을 1월 2일에 완료하면 1월 2일 잔디에 반영됨 (깃허브가 커밋한 날 기준으로 잔디를 채우는 것과 동일한 원리)
- 잔디를 클릭하면 해당 날짜에 어떤 활동을 했는지 상세 보기 가능
- 스트릭 프리즈: ❌ 제외
- 태그별 레벨 시스템: ❌ 제외

---

## 7. 목표 & 통계 ✅

- 오늘 기준 체크리스트 달성률(%) 시각화
- 시간 목표가 설정되어 있으면, 목표 대비 실제 기록된 시간 비율(%) 시각화
- "아직 끝내지 못한 일" 섹션을 별도로 운영 — 과거에 시간대를 정해 계획했지만 완료하지 못한 블록/활동을 모아두는 곳 ("계획 vs 실제"의 실패 기록)
  - 사용자가 완료 체크하거나 삭제하면 목록에서 사라짐

---

## 8. 반복 & 템플릿 ✅

> 용어 구분: 여기서 말하는 "템플릿"은 **하루/주 단위 일정 전체를 복제**하는 기능. 1번 항목의 "블록 템플릿"(재사용 가능한 단일 활동 정의)과는 다른 개념.

- 반복 설정 기능 (매일/매주 반복되는 블록)
- 하루/주 전체를 템플릿으로 복제하는 기능
- 편의성을 위해 **둘 다 도입**

---

## 9. 일정 관리 ✅

- 블록끼리 시간이 겹치면 경고 표시
- 블록의 시각적 크기는 설정된 시간 길이에 비례해서 캘린더에 표시 (예: 2시간 블록이 1시간 블록보다 시각적으로 더 크게)
- **블록 가장자리를 잡고 드래그하면 시작/종료 시간(시간 범위)을 직접 조정 가능** — 시간 입력은 별도 피커 없이 오직 이 방식으로만 (1번 항목 참고)
- 습관 스태킹: 블록끼리 "이거 끝나면 이거" 식으로 연결해서 루틴 체인 구성 — 도입
- 구글 캘린더 연동: ❌ 제외
- 아이젠하워 매트릭스 뷰: ⏸ 보류

---

## 10. 회고 & 기록 ✅

- 활동(블록) 단위로 구체적인 메모 작성 가능
  - 예: "운영체제 공부" 블록 → "7장 페이징, 8장 가상메모리 정리함" 같은 세부 내용 기록
- ~~오늘의 기분을 하루 단위로 이모지로 간단히 표현~~ → ❌ 제외 (도입했다가 제거)

---

## 11. 마감 작업 (데드라인 태스크) ✅

- 특정 시간대 없이 **날짜(마감일)만 지정**하는 작업 타입 (예: "기말 레포트 — 6월 30일까지")
- 캘린더 뷰에 마감일에 마커로 표시
- "아직 끝내지 못한 일"(7번 항목)과는 **별개의 독립적인 개념**:
  - 아직 끝내지 못한 일 = 과거에 계획했는데 못 끝낸 것 (실패 기록)
  - 마감 작업 = 애초에 시간대를 정한 적 없이 "이 날짜까지 끝내면 되는" 일
- 마감 작업은 자체적으로 상태를 가짐: **진행중 / 마감 지남(미완료)**
  - 마감일이 지났는데 미완료면, 7번 섹션으로 합쳐지지 않고 마감 작업 목록 안에서 "지난 마감"으로 별도 표시 (예: 빨간색 강조)
- 사용자가 완료 체크하거나 삭제하면 목록에서 사라짐
- 필요하면 사용자가 직접 시간대를 지정해서 일반 타임블록으로 전환 가능 (마감 작업 → 타임블록)
